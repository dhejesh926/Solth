Solth — A Tiny Toy Language

What is Solth?

Solth is a tiny, easy-to-read interpreted toy language. It is my third programming language and my first serious language project that I am publishing publicly.

Solth is intentionally minimal. It does not try to be fast, safe, or complete. Instead, it focuses on being simple, hackable, and understandable.

A programmer can learn Solth in under 5 minutes by reading the demo file. The source code is small and easy to modify.

Why Lua?

Solth is implemented in Lua (LuaJIT) because Lua is the interpreted language I use the most. Lua is simple, expressive, and perfect for writing small interpreters.

The Solth interpreter written in Lua is compact and readable, making experimentation and modification very easy.

What is the role of C?

Lua by itself does not provide a standalone executable interpreter with proper command-line argument handling.

To solve this, Solth uses a C host that embeds LuaJIT. This C program loads solth.lua and executes .solth files.

This makes Solth feel like a real executable language interpreter.

Supported Features

  • Variables (dynamically typed)
  • Input and output
  • if and else conditionals
  • Comparisons
  • Function declarations
  • Calling functions
  • Nested functions
  • Expression evaluation

There are no data types — everything is dynamic.

Unsupported / Broken Features

  • elseif is broken
  • Loops are not implemented
  • Function names cannot contain underscores (_)
  • Function arguments are not supported
  • Many edge cases are not handled

These limitations are intentional for learning purposes.

How to Learn Solth

Read the demo.solth file.

That is enough to understand the entire language.

How to Build Solth

Requirements:

  • LuaJIT headers and libraries installed

Steps:

  • Run make
  • A statically linked Solth interpreter will be produced

IMPORTANT:
Do not remove solth.lua. The interpreter depends on it at runtime.

Project Philosophy

Solth is not meant to compete with real languages. It exists to explore language design, interpreters, and embedding.

It is small, imperfect, and educational.
